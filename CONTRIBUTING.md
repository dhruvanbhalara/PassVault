### 1. Reporting Bugs
If you find a bug, please open an issue with:
- A clear description of the problem.
- Steps to reproduce the issue.
- Details about your environment (device, OS, Flutter version).

### 2. Suggesting Features
Have an idea to make PassVault better? Open an issue and label it as a `feature request`.

### 3. Submitting Pull Requests (PRs)
- **Fork** the repository and create your branch from `main`.
- Ensure your code follows the existing style (run `dart format .`).
- Verify that all tests pass (`flutter test`).
- Document any new features or changes.
- Open a PR with a clear description of what your changes accomplish.
